## Technical Interview Quick Facts

### Python/Django Key Points
```
‚úì GIL affects multi-threading (use multiprocessing for CPU tasks)
‚úì Use ORM to prevent SQL injection
‚úì select_related() for ForeignKey/OneToOne
‚úì prefetch_related() for ManyToMany/Reverse FK
‚úì Add indexes on frequently queried columns
‚úì Use signals for event-driven logic
‚úì Always validate input
‚úì Use transactions for data consistency
```

### Django REST Framework
```
‚úì Serializers: Convert models to JSON
‚úì ViewSets: Automated CRUD operations
‚úì Permissions: Control who can access
‚úì Authentication: Verify who they are (JWT, Token, Session)
‚úì Throttling: Rate limiting
‚úì Pagination: Break large results into pages
‚úì Filtering: Allow users to filter results
‚úì Status codes: 200/201 (success), 400 (bad request), 401 (unauth), 403 (forbidden), 404 (not found)
```

### Performance Optimization
```
‚úì Profile before optimizing
‚úì Use Django Debug Toolbar to find N+1 queries
‚úì Add database indexes
‚úì Implement caching (Redis)
‚úì Use pagination for large lists
‚úì Optimize queries with select_related/prefetch_related
‚úì Monitor response times
‚úì Use Gunicorn + Nginx for production
```

### Security
```
‚úì Never concatenate user input into queries
‚úì Use parameterized queries or ORM
‚úì Hash passwords (bcrypt)
‚úì Use HTTPS/SSL
‚úì Validate all inputs
‚úì Implement CORS properly
‚úì Use rate limiting
‚úì Secure headers (CSP, HSTS, etc.)
```

- [ ] Arrive at location

- [ ] Don't interrupt
- [ ] Make eye contact while listening

- [ ] Speak clearly and confidently
- [ ] Use gestures naturally
- [ ] Pause before answering complex questions

- [ ] Thank them for their time
- [ ] Firm handshake
- [ ] Smile

- [ ] Mention specific topics discussed
- [ ] Reiterate your interest
- [ ] Keep it to 3-4 paragraphs
